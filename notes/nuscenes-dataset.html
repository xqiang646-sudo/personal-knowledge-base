读懂 nuScenes 数据集

nuScenes 不是“一堆图片和点云文件”，而是一套由传感器文件、关系型元数据、标定、车辆位姿、地图、标注与评测协议共同组成的多传感器时序数据库。

贡献角色：导师原创整理　对象：多传感器感知入门学习者　证据状态：基于 nuScenes v1.0 官方网站、论文、schema 与 devkit

使用边界：下载、使用和再分发数据前，应阅读并遵守 nuScenes 当前使用条款；mini 集只适合熟悉数据和调通代码，不能支撑正式模型结论。

数据集定位数据关系文件与时间坐标边界类别与评测最小练习常见误区资料

先理解它解决什么问题

nuScenes 面向自动驾驶多模态研究。官方 v1.0 数据包含 1000 个约 20 秒的城市驾驶场景，采集于波士顿和新加坡；传感器套件包括 6 个相机、5 个毫米波雷达、1 个激光雷达以及 GPS/IMU。

学习重点不是背数据规模，而是能从一条记录沿 token 找到真实传感器文件，并清楚说出它对应的时间、标定、自车位姿、标注和坐标系。

核心：它是一张由 token 连接的数据图

scene
  ├─ log ── location / map
  └─ sample ── prev / next
       ├─ sample_data ── sensor file
       │    ├─ calibrated_sensor ── sensor → ego 外参、相机内参
       │    └─ ego_pose ── ego → global 位姿
       └─ sample_annotation
            ├─ instance
            └─ category / attribute / visibility
先分清逻辑时刻、传感器帧和标注属于哪一层
记录表示什么关键连接
scene从一段 log 中抽取的连续场景首尾 sample、log、场景内 sample 数量
sample2 Hz 的多传感器标注关键时刻，不是磁盘文件prev/next、各通道 sample_data、annotation
sample_data某个传感器在某一时刻的一份真实数据文件、时间戳、是否关键帧、标定、自车位姿
calibrated_sensor传感器相对 ego body frame 的标定外参；相机记录内参
ego_pose该传感器时刻的 ego 相对地图 global frame 位姿平移、四元数、时间戳
sample_annotation某个 sample 上一个目标的 3D 标注instance、类别、属性、可见性、前后标注

token 的意义：每条记录用 token 唯一标识，*_token 像数据库外键一样连接其他表。理解模型代码前，先手工走通一次这张关系图。

目录名不能替代时间关系

<NUSCENES_ROOT>/
├── samples/       # 关键帧传感器数据
├── sweeps/        # 关键帧之间的中间传感器数据
├── maps/          # 地图数据
└── v1.0-mini/     # JSON 元数据；也可能是 trainval / test
  • sample 是一次逻辑关键时刻；sample_data 才对应相机图像、点云或雷达文件。
  • 不同传感器有各自时间戳，不能默认完全同步；多帧融合要按真实时间和自车位姿补偿。
  • is_key_frame=false 的 sweep 仍有自己的时间戳、ego pose 和 calibration 引用。
  • 一个 20 秒 scene 通常约有 40 个 sample，但程序应读取 scene["nbr_samples"]，不要写死。

先画清坐标链，再搬点和框

sensor frame
→
ego frame at sensor time
→
global frame of map
  • calibrated_sensor 提供 sensor 到 ego 的外参；ego_pose 提供该数据时刻 ego 到 global 的位姿。
  • 原始 sample_annotation 的位置和旋转位于 global frame；devkit 的 get_sample_data() 会把 box 转到当前 sensor frame。
  • 标注尺寸顺序是 width, length, height；记录中的四元数顺序是 w, x, y, z。
  • 把两个传感器或两个时刻的数据融合时，必须同时处理外参、ego motion 与各自时间戳。

最常见的双重变换：先用 get_sample_data() 得到 sensor-frame box，又把它当 global box 再执行一次 global→sensor。每次变换前都要写出“当前坐标系”和“目标坐标系”。

数据类别与评测类别不是同一个概念

基础 taxonomy、任务输出和评测协议各有自己的边界
概念官方 v1.0 规则用途
论文中的 3D box taxonomy23 类，另有 8 种属性描述原始 3D 目标标注
Detection benchmark合并和过滤后使用 10 个检测类提交检测结果并统一评测
nuScenes-lidarseg32 个通用语义类，挑战合并为 16 类逐点激光雷达语义分割
关键帧sample 为 2 Hz 标注时刻任务结果按 sample 组织
NDS综合 mAP 与多项真实误差不能只看一个 IoU 或单次可视化

“23 类、10 类、32 类、16 类”来自不同标签空间和任务，不能混用。模型能输出 box，也不等于结果可评测；类别映射、global 坐标、速度、属性、结果 JSON 和官方 split 都必须满足对应任务协议。

最小练习：沿 token 走一遍数据图

from nuscenes.nuscenes import NuScenes

nusc = NuScenes(
    version="v1.0-mini",
    dataroot="<NUSCENES_ROOT>",
    verbose=False,
)

scene = nusc.scene[0]
sample = nusc.get("sample", scene["first_sample_token"])

for channel, sd_token in sample["data"].items():
    sd = nusc.get("sample_data", sd_token)
    calib = nusc.get("calibrated_sensor", sd["calibrated_sensor_token"])
    sensor = nusc.get("sensor", calib["sensor_token"])
    pose = nusc.get("ego_pose", sd["ego_pose_token"])

    print(channel, sensor["modality"], sd["timestamp"],
          sd["is_key_frame"], sd["filename"], pose["translation"])

ann = nusc.get("sample_annotation", sample["anns"][0])
print("box in global:", ann["translation"], ann["size"], ann["rotation"])

验收标准：能画出代码经过的表关系；能比较各通道时间戳；能解释原始 annotation 与传感器画面中 box 的坐标差异；能沿 sample["next"] 遍历 scene，而不是假设固定帧数。

常见误区

“sample 就是一帧图片”

它是多传感器标注时刻；每份真实文件属于 sample_data。

“所有通道完全同步”

应比较各自 timestamp；跨时刻融合还要补偿 ego motion。

“annotation 默认在 LiDAR 坐标”

原始 annotation 位于 global；SDK 不同接口可能返回不同坐标系。

“mini 指标高说明模型有效”

mini 用于学习与集成调试，正式结论需官方 split 和评测协议。

“instance 能跨 scene 追踪”

官方 schema 明确 instance identity 不跨 scene 保留。

“公开下载等于可随意商用”

必须查看当前 Dataset Terms，不自行扩大使用与再分发权限。
