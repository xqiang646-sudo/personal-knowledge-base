自动驾驶系统总览

自动驾驶的根本任务，是从带噪声、延迟和不完整的传感器测量中推断真实环境，再持续选择安全且可执行的车辆动作。

第一性原理核心模块行人示例闭环与安全学习重点

第一性原理：测量、估计、决策、执行

真实世界
→
传感器测量
→
环境与自车状态估计
→
规划安全动作
→
车辆执行

相机只给出像素，激光雷达只给出空间反射点，GNSS/IMU 只给出位置和运动测量。软件需要把这些原始数据转换成车辆、行人、道路、信号灯、自车位置和未来轨迹等结构化信息。

关键认识：自动驾驶不是一个模型完成所有任务，而是多模块通过时间戳、坐标系和 ROS 2 接口协同工作的实时闭环系统。

核心模块分别回答什么问题

Sensing 传感器处理

数据格式对不对、时间是否同步、坐标是否正确、噪声是否可控？

Localization 定位

自车在地图中的位置、姿态、速度和角速度是什么？

Map 地图

道路在哪里、车道如何连接、限速和交通规则是什么？

Perception 感知

周围有什么目标，它们在哪里、朝哪儿、以多快速度运动？

Prediction 预测

车辆、行人和骑行者未来几秒可能怎样运动？

Planning 规划

应该走哪条道路、采取什么行为、生成怎样的安全轨迹？

Control 控制

方向盘角度、加速度和制动命令应该是多少？

Vehicle Interface

怎样把通用控制命令转换为底盘能理解的 CAN 报文？

System 系统安全

节点、传感器和硬件是否健康；故障时如何降级或停车？

Visualization / HMI

RViz 帮工程师调试算法，HMI 帮操作员启停任务和观察关键状态。

示例：车辆发现前方行人

  1. 相机采集图像，激光雷达采集点云，组合导航提供车辆姿态和时间基准。
  2. ADCU 或驱动节点接入数据，完成解串、同步、畸变校正、去噪和坐标转换。
  3. YOLOX、BEVFusion 或点云检测器识别行人，并输出类别、位置、尺寸和置信度。
  4. 跟踪模块关联连续帧，估计行人速度；预测模块推断可能的未来路径。
  5. 行为规划判断需要减速或停车，运动规划生成满足车辆约束的轨迹。
  6. 控制器计算转向和制动量，经车辆接口和 CAN 总线发送给 VCU。
  7. 底盘执行制动并反馈实际车速、转角和故障状态，系统进入下一次闭环计算。

为什么必须是闭环

规划命令不等于车辆已经准确执行。路面附着、载荷、转向机构和通信延迟都会导致实际状态偏离目标，所以系统必须持续读取反馈并重新计算。

实时性

不仅看平均延迟，还要关注最大值、P95/P99、丢帧和抖动。

一致性

图像、点云、定位和地图必须处于正确时间和正确坐标系。

安全性

故障监控、急停、最小风险动作和人工接管必须独立考虑。

感知实习生最先掌握的内容

  1. 坐标系与 TF：能分清 sensor frame、camera optical frame、base_link、odom 和 map。
  2. 时间戳与同步：确认检测框、图像和点云是否属于同一时刻。
  3. 消息流水线：逐级观察 raw、preprocessed、detected、tracked、predicted 和 trajectory。
  4. 标定优先级：融合偏移先查同步、内外参、TF 和预处理，再怀疑模型。
  5. 输入—处理—输出：看到任何节点，都能说清输入是什么、解决什么问题、输出给谁。
