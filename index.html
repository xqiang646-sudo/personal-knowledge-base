INTERN LEARNING PATH · 从零开始

从看懂一个名词，走到完成一次可复查的工程交付

默认学习者不会 Linux、Python、ROS 2、线性代数和自动驾驶。路线先补最小基础，再用 16 门课程串起系统、数据、几何、运行时、部署和验证。

先对齐边界

“学完”到底意味着什么

这条路线培养的是受限范围内的独立学习与工程交付能力，不把读过文章误写成生产权限或实车验收资格。

默认不假设

不预设终端、Git、Python、ROS 2、矩阵、导数、数据集或车辆控制经验。

结业可以做到

完成小型数据任务、离线分析、只读排障和带环境边界的证据化交付。

结业不自动获得

真实车辆写操作、生产发布、正式验收和扩大测试范围仍需要明确授权。

学习闭环

每一课都按五步完成，不以“看完页面”为终点

  1. 1诊断

    先做先修小题，会的跳过，不会的补齐。

  2. 2理解

    用自己的话解释对象、输入、输出和假设。

  3. 3动手

    手算、写脚本或在沙箱执行只读检查。

  4. 4留证

    保存命令、输入、结果、版本、失败样本和边界。

  5. 5过门

    达到阶段验收后再进入下一阶段。

带教原则：学习者必须能解释、能做一个新例子、能指出失效条件并给出证据；背术语、背公式、背命令都不等于掌握。

唯一主线

零基础到工程闭环的 7 个阶段

建议按顺序完成。已有基础时可以通过阶段诊断跳过局部内容，但阶段交付和验收不能跳过。

0 预备 1 系统 2 工具 3 数据 4 Autoware 5 车辆 6 交付
  1. 阶段 0

    零基础预备

    先补共同语言和安全边界，避免后面每个术语都成为断点。

    建议 18–24 小时

    阶段门：能在沙箱运行脚本、解释命令输出和退出码，并区分只读与写操作。
    01必修可诊断跳过

    零基础预备课

    计算机、Python、数学、时间与网络、ROS 词汇、机器学习统计和 CAN 数字基础。

    交付：基础诊断表与六组最小练习。 开始第一课 →
  2. 阶段 1

    建立系统全景

    先知道整车系统为什么存在，再认识硬件和 CPU/GPU 数据通路。

    建议 6–8 小时

    阶段门：独立画出传感器到车辆执行的闭环，并标注每段数据和责任模块。
    02系统

    自动驾驶系统总览

    从测量、估计、决策和执行理解感知、定位、规划、控制与安全闭环。

    交付：一张系统闭环图。 进入课程 →
    03硬件网络

    车辆硬件与车载网络

    分清传感器、组合导航、IPC、VCU、交换机、路由器和常见接口。

    交付：硬件角色与数据链图。 进入课程 →
    04GPU 图解

    GPU 工作流程图

    看懂 CPU 预处理、显存、并行计算、模型推理和感知输出之间的分工。

    交付：CPU/GPU 分工说明。 进入课程 →
  3. 阶段 2

    掌握工程工具与证据

    会操作不等于会判断。先学安全地取得证据，再提出工程结论。

    建议 8–12 小时

    阶段门：为一个虚构故障提交包含主机、版本、配置、时间和原始输出的证据卡。
    05LinuxGit

    远程运维与开发工具

    学习 SSH、SCP、NoMachine、VS Code Remote、Git、目录、网络、日志和服务。

    交付：沙箱操作记录与主机边界说明。 进入课程 →
    06M01方法

    证据优先的工程排障

    区分源码、部署、运行和验收事实，先做只读采样，再建立假设。

    交付：一张可复查证据卡。 进入课程 →
  4. 阶段 3

    理解坐标、感知与数据任务

    先建立空间坐标，再把点云、地图、模型和 nuScenes 数据关系连起来。

    建议 28–36 小时

    阶段门：交付一份可追溯 mini 数据包，所有点、框、位姿和统计都声明坐标系、时间和分母。
    07M06数学

    从线性代数到坐标系转换

    从向量、基和换基推导旋转、平移、齐次变换、四元数与传感器坐标链。

    交付：手算例题、变换脚本和往返测试。 进入课程 →
    08感知地图

    感知算法与地图基础

    认识点云、地图、检测、融合、跟踪、预测、YOLOX、BEVFusion、训练与推理。

    交付：感知输入—处理—输出契约图。 进入课程 →
    09M05数据任务

    nuScenes 数据集与数据任务入门

    完成盘点、索引、QC、scene split、标签合同、统计、可视化和交付。

    交付：inventory、manifests、QC 与 dataset card。 进入课程 →
  5. 阶段 4

    进入 Autoware 与 ROS 2 运行时

    理解规划控制的职责，再沿 ROS 2 图谱证明数据在哪一段失效。

    建议 14–18 小时

    阶段门：在合成或离线环境完成一次只读断链分析，结论能指向具体发布者、契约或消费者。
    10规划Autoware

    规划控制与 Autoware

    分清 route、scenario、behavior、path、trajectory、control 和主要功能域。

    交付：规划控制分层图与消息语义表。 进入课程 →
    11M02ROS 2

    沿 ROS 2 图谱排查数据链

    从进程、节点、话题、QoS、频率、内容、时间和 TF 逐层查找断点。

    交付：一份可复现的断链定位报告。 进入课程 →
  6. 阶段 5

    理解车辆运动与控制接口

    从无侧滑和曲率走到轨迹、转角和离线 CAN 信号，不在真实车辆上试发命令。

    建议 20–24 小时

    阶段门：通过直线、圆、左右转、倒车、噪声和离线报文测试，并解释模型失效条件。
    12M07车辆模型

    阿克曼转向与自行车模型

    第一遍学到自行车模型；完成下一课曲率后，再回来学习倒车、积分和质心模型。

    交付：公式推导与小型轨迹仿真。 进入课程 →
    13M08曲率

    从微积分和线性代数推导曲率

    从弧长、导数和行列式推导连续曲率、三点曲率和航向差分。

    交付：曲率实现、数值实验和边界测试。 进入课程 →
    14CAN仅离线

    CAN 总线接口功能解析

    理解报文、接线、终端电阻、DBC、RViz/HMI 和车辆接口的边界。

    交付：离线帧解码与控制信号映射。 进入课程 →
  7. 阶段 6

    把结果变成可交付结论

    最后统一训练/部署契约、性能口径、回归证据和正式结论边界。

    建议 16–24 小时

    结业门：另一位工程师能按文档复现结果，失败、未执行项和目标环境边界没有被隐藏。
    15M03部署

    训练到部署的契约

    对齐数据、几何、语义、运行时和性能，避免“引擎能跑”被误写成结果等价。

    交付：训练/部署 manifest 与 golden sample 记录。 进入课程 →
    16M04结业

    从候选结果到正式结论

    定义环境、场景、指标、例外、回归和交接，把一次成功升级成可复查结论。

    交付：结业项目的完整证据包与复现说明。 进入最后一课 →

主线完成后

按任务方向加深，不重新开一套基础课

数据 / 感知

数据质量与模型输入

深做坐标、nuScenes、感知、GPU 与部署契约，交付数据卡、QC、可视化和模型输入 manifest。

ROS / 集成

数据链与系统排障

深做工程工具、证据卡、Autoware、ROS 2 和只读 CAN 诊断，交付可复现的断链报告。

规划 / 控制

几何、轨迹与车辆接口

深做坐标、阿克曼、曲率、规划控制和离线 CAN 映射，交付带边界测试的几何工具。

部署 / 验证

运行时契约与回归

深做 GPU、部署契约、证据方法和验证闭环，交付环境矩阵、golden samples 与回归报告。
